# SegPay — Multi-Currency & Languages

> **Load this when:** configuring packages, discussing pricing localization, or
> debating which currency to advertise on the pricing page.

## Dynamic Multi-Currency (DMC)

SegPay supports **15 currencies** on the hosted payment page, identified by
three-letter ISO codes:

USD, EUR, GBP, CAD, AUD, JPY, CHF, and others (full list in the Merchant
Portal).

### Two configuration modes

1. **Geo IP detection** — SegPay detects the consumer's country from IP and
   displays the matching currency. If the detected currency is not supported, it
   falls back to the price point's assigned currency.
2. **Price Point Currency** — always display the price in the currency assigned
   to the price point, regardless of consumer location.

### Per-link override

Append `&dmcurrency=<iso>` to the Signup URL to force an initial currency:

```
...&dmcurrency=hkd        → displays in HKD
...&dmcurrency=eur        → displays in EUR
```

The consumer can still change the currency via the dropdown at the top of the
payment page — the override only sets the **initial** display.

### Activation

Merchant Portal → **My Websites → Manage Packages** → edit package →
**MultiCurrency Options** dropdown.

### Reporting implication

> **⚠️ Important**: "When viewing data about DMC transactions, amounts are
> reported in your base currencies."

If Miximodel's base currency is EUR but a Canadian customer pays in CAD,
transaction reports show the **EUR equivalent**, not the CAD amount charged.
Plan dashboards and analytics accordingly — do not mix base-currency amounts
with display-currency amounts.

### Miximodel recommendation

- **Enable Geo IP detection** for the Pro plan package.
- **Base currency**: EUR (Miximodel is EU-based; simplifies accounting and tax
  reporting).
- **Supported display currencies**: enable all 15 at start; there is no cost to
  supporting more, and DMC smoothes abandonment from international users.
- **Advertise prices in EUR** on the pricing page; the checkout will
  auto-localize.

## Languages

SegPay payment pages support **21 languages**:

Czech, Danish, Dutch, English, French, German, Greek, Hebrew, Hindi, Italian,
Japanese, Norwegian, Portuguese, Russian, Simplified Chinese, Slovak, Slovenian,
Spanish, Swedish, Traditional Chinese.

### Default behavior

Browser language is auto-detected. **No configuration needed** in the common
case.

### Per-link override

Append `&paypagelanguage=<ISO two-letter>` to force a language:

```
...&paypagelanguage=FR    → French
...&paypagelanguage=NL    → Dutch
...&paypagelanguage=DE    → German
```

Consumers can still switch language via the dropdown at the top of the payment
page.

### Miximodel recommendation

Do **not** force a language. Miximodel already runs an i18n layer (React /
Inertia locale). When building the Signup URL, pass the user's current Miximodel
locale as the initial override to keep the language consistent across the
redirect boundary:

```ts
const paypagelanguage = user.locale.toUpperCase(); // 'FR', 'EN', ...
```

Fall back to browser auto-detection (no param) if the user is anonymous.

## Tax Considerations — OPEN QUESTION

**⚠️ Public documentation does not state** whether SegPay handles EU VAT / UK
VAT / US sales tax collection and remittance.

This is a **material business concern**:

- Lemon Squeezy was a **merchant of record (MoR)** — it collected VAT
  automatically and remitted it on behalf of merchants. That was its killer
  feature for EU merchants.
- SegPay's positioning suggests it operates as a **payment processor**, not a
  MoR — meaning Miximodel would be responsible for VAT collection, registration
  (OSS / MOSS), and remittance.

**Before going live**, confirm with the SegPay account manager:

1. Does SegPay act as merchant of record?
2. Does SegPay collect VAT automatically based on consumer country?
3. Does SegPay provide tax reports for MOSS / OSS filing?
4. If not, what is the recommended workflow — third-party service (Quaderno,
   Taxamo, Stripe Tax) or in-house logic?

This answer **directly impacts the pricing page** (prices inclusive or exclusive
of tax?), the invoice template, and the Miximodel cost structure.

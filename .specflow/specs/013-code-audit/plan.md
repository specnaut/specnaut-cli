# Implementation Plan: High-altitude multi-seat parallel audit (`/code-audit`)

**Branch**: `013-code-audit` | **Date**: 2026-06-13 | **Spec**: [spec.md](./spec.md) **Input**:
Feature specification from `.specflow/specs/013-code-audit/spec.md` (issue mkrlabs/specflow#379,
epic mkrlabs/specflow-monorepo#12)

## Summary

Add a bundled `/code-audit` orchestrator skill plus a `collect-audit-scope.sh` scope-collection
script. The script resolves an audit scope on `main` (path/range/unpushed/since-tag/last-N) and
emits CATEGORY SIGNALS (file counts per category). The skill reads the signals, dispatches the
applicable auditor seats (architecture / security / performance / dependency / a11y) IN PARALLEL,
then synthesizes one deduplicated, severity-ranked report closing with an aggregated
`REVIEW SUMMARY` (dominance rule: fail > needs_followup > pass; counts summed). Read-only;
complements `/specflow audit <axis>`.

Two surfaces, both content + a shell script — no runtime/domain code:

1. `templates/core/skills/code-audit/SKILL.md` +
   `templates/core/skills/code-audit/scripts/collect-audit-scope.sh`.
2. Bundle distribution: register in `templates/manifest.json`, regenerate `src/templates_bundle.ts`,
   mirror to `plugin/`, add bundle-inclusion + script-behaviour tests.

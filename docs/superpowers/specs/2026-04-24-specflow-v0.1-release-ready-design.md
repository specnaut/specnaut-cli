# Specflow v0.1 release-ready bundle — Design

**Date** : 2026-04-24 **Status** : draft, awaiting review **Scope** : 3 sub-bricks packaged together
to make Specflow publicly distributable **Prerequisites** : v0.1-init + backlog-sync merged on
`main` (126 tests green)

---

## 1. Objectives and non-objectives

### Objectives

Deliver three short additions that, together, transform Specflow from "works on Kevin's machine" to
"installable and debuggable by anyone" :

1. **`specflow check`** — diagnostic of environment and existing Specflow installation
   (prerequisites + project health).
2. **`--force` on `specflow init`** — allow re-scaffolding a project with `.claude/` or `.specify/`
   already present, saving existing files before overwrite.
3. **`install.sh` + Homebrew formula** — two reproducible installation channels for the binary.

### Non-objectives

- `specflow upgrade` (3-way merge template diff) — deferred to next sub-brick.
- Auto-update after successful `self-update` (reboot/relaunch) — not requested.
- Multi-tap Homebrew (core, official) — we provide only a personal tap `kevinraimbaud/homebrew-tap`.
- Binary signature/notarization (macOS Gatekeeper) — deferred to v0.2+.

## 2. Sub-brick 1 — `specflow check`

### CLI surface

```
specflow check [--project]
```

Without arguments : checks the **environment** (external binaries + versions). With `--project` :
additionally, checks the **Specflow installation** of cwd (files present, valid config, coherent
template version).

### Output

Compact ASCII table, exit 0 if all OK, exit 1 otherwise :

```
specflow check — environment

  git         ✓ git version 2.43.0
  gh          ✓ gh version 2.50.0 (authenticated as kevinraimbaud)
  deno        ⚠ deno not in PATH (optional — only needed for self-update on non-compiled runs)

specflow check — project (./)

  .specify/               ✓ present
  .claude/                ✓ present
  constitution            ✓ .specify/memory/constitution.md (not empty)
  backlog config          ✓ .specflow/config.yml (github:kevinraimbaud/specflow → project #3)
  templates version       ✓ matches binary (0.2.0)

All checks passed.
```

### Architecture (continued DDD)

- **Domain** : `src/domain/check_result.ts` — `CheckResult`, `CheckOutcome` value types (pure).
- **Application** : `src/application/run_checks.ts` — `RunChecksUseCase` (orchestrates N independent
  checks through ports).
- **Application ports additions** (`ports.ts`) :
  - `EnvironmentProbe` — lookups for `git`, `gh`, `deno` versions + `gh auth status`.
  - `ProjectInspector` — checks file presence, reads constitution emptiness, reads `config.yml`,
    compares `TEMPLATES_VERSION`.
- **Infrastructure** :
  - `DenoEnvironmentProbe` — shells out via `SubprocessRunner` (already exists).
  - `FsProjectInspector` — reads filesystem via `Deno.stat`/`readTextFile`.
- **CLI handler** : `src/cli/handlers/check_handler.ts` — renders the table.

### Tests

- `tests/application/run_checks_test.ts` — 4 tests (all pass / mixed / project mode / project mode
  with missing files). Inject fake probes.
- `tests/infrastructure/fs_project_inspector_test.ts` — 3 tests against tmp dirs (well-formed
  project, missing .specify, old templates version).

## 3. Sub-brick 2 — `specflow init --force`

### CLI surface

Extension of existing parser : add `--force` to the boolean flag set of the `init` intent. Already
reserved in v0.1 spec (§6 of initial design), now implemented.

### Semantics

1. Without `--force` : current behavior (abort on conflict).
2. With `--force` :
   - For each managed file that already exists, rename it to `{path}.specflow.bak` (fixed suffix —
     no timestamp to remain deterministic). If `{path}.specflow.bak` already exists from a previous
     run, overwrite it (the rule "lose nothing" is assumed by the explicit name `.specflow.bak`,
     it's up to the user to preserve their changes).
   - Then write the template over it as usual.
   - If a `.specflow.bak` is written, log it :
     `↳ backed up existing .claude/commands/speckit.specify.md → .claude/commands/speckit.specify.md.specflow.bak`.
3. Files **not managed** by Specflow (outside the manifest) remain intact, even with `--force`.

### Architecture

- **Modified existing port** : `FsWriter.writeBundle(bundle, targetDir, { overwrite: true })`
  already signals it can overwrite. We need to add an option `{ backupExisting: true }` that does
  the rename before each write and returns the list of backups performed.
- **Domain unchanged**.
- **CLI parser** : add `"force"` to boolean set in parser.ts, propagate in `InitIntent`.
- **Init handler** : if `intent.force`, pass `backupExisting: true`.
- **`InitProjectUseCase`** : if `force=true`, skip the `detectConflicts` gate (since we accept the
  overwrite) and pass `{ overwrite: true, backupExisting: true }` to the writer. Expose
  `backups: string[]` in `InitResult` so the handler displays them.

### Tests

- Unit `DenoFsWriter` : 2 tests (backup creates .specflow.bak; existing .specflow.bak is
  overwritten).
- Integration `specflow init --force` in tmp dir with pre-existing `.claude/` — exit 0,
  `.specflow.bak` files present, original files replaced.

## 4. Sub-brick 3 — `install.sh` + Homebrew formula

### 4.1 `install.sh`

Script `curl | sh`-friendly. Detects OS/arch, downloads the right binary from latest release,
verifies SHA256, installs in `$PREFIX` (default `/usr/local/bin`).

Usage :

```bash
curl -fsSL https://raw.githubusercontent.com/kevinraimbaud/specflow/main/install.sh | bash
# or: pin a version
curl -fsSL https://raw.githubusercontent.com/kevinraimbaud/specflow/main/install.sh | VERSION=v0.1.0-alpha.1 bash
# or: install to a different prefix
curl -fsSL https://raw.githubusercontent.com/kevinraimbaud/specflow/main/install.sh | PREFIX=$HOME/.local/bin bash
```

Behavior :

1. Detects `uname -s` + `uname -m` → maps to `specflow-<os>-<arch>[.exe]`.
2. If `VERSION` not provided, query
   `https://api.github.com/repos/kevinraimbaud/specflow/releases/latest`.
3. Download `specflow-<triple>` + `specflow-<triple>.sha256` via `curl -fsSL`.
4. Verify `shasum -a 256 -c` (or `sha256sum` on Linux).
5. Chmod 755 + move to `$PREFIX/specflow`. If `$PREFIX` requires sudo, prompt (but don't auto-sudo).
6. Print `specflow --version` for confirmation.

Robustness :

- `set -euo pipefail` at top.
- Explicit errors if curl/shasum missing.
- Unsupported OS/arch → list known targets.
- Windows : refuse with instruction to manually download from GH Releases (Windows shell script out
  of scope).

### 4.2 Homebrew formula

Separate tap repository : `kevinraimbaud/homebrew-tap`. Formula file : `Formula/specflow.rb`.

```ruby
class Specflow < Formula
  desc "Spec-driven CLI with auto-chain, review phase, and backlog sync"
  homepage "https://github.com/kevinraimbaud/specflow"
  version "0.1.0-alpha.1"

  on_macos do
    on_arm do
      url "https://github.com/kevinraimbaud/specflow/releases/download/v#{version}/specflow-macos-arm64"
      sha256 "..." # populated at release
    end
    on_intel do
      url "https://github.com/kevinraimbaud/specflow/releases/download/v#{version}/specflow-macos-x64"
      sha256 "..."
    end
  end

  on_linux do
    on_arm do
      url "https://github.com/kevinraimbaud/specflow/releases/download/v#{version}/specflow-linux-arm64"
      sha256 "..."
    end
    on_intel do
      url "https://github.com/kevinraimbaud/specflow/releases/download/v#{version}/specflow-linux-x64"
      sha256 "..."
    end
  end

  def install
    bin.install Dir["*"].first => "specflow"
  end

  test do
    assert_match "specflow #{version}", shell_output("#{bin}/specflow --version")
  end
end
```

**Automation**: extend `.github/workflows/release.yml` to push the formula file to
`kevinraimbaud/homebrew-tap` after a successful release. v0.1 of this sub-brick = **formula file
committed in the Specflow repo** (`packaging/homebrew/specflow.rb` template, `.sha256` values
placeholders updated by the release workflow). **Auto-publish to the tap is deferred** — we just
describe how to do it manually.

### Tests

- `install.sh` → shellcheck clean + `install.sh --dry-run` option that skips download/move and
  prints what it WOULD DO (for tests).
- Homebrew formula : `brew audit --strict --online` (skip in CI, docs only).

## 5. Impact on binary and permissions

- `specflow check` requires `--allow-run=git,gh,deno` and `--allow-read`. Already baked.
- `specflow init --force` : no new permissions (just rename + write via Deno.rename/writeFile,
  already OK).
- `install.sh` : pure bash, no Deno. Independent of binary.
- Homebrew formula : no Deno either.

No changes to `scripts/build.ts` or `.github/workflows/release.yml` required for permissions. The
release workflow could be extended to also upload `install.sh` alongside the binaries and commit to
the tap, but that's an optional extension.

## 7. Risks and open questions

- **`install.sh` signature** : no macOS notarization in v0.1. User will need to
  `xattr -d com.apple.quarantine` or go through System Settings on first execution. Document clearly
  in script + README.
- **Homebrew auto-publish** : formally deferred. Until then, release process = 4 manual steps (see
  existing `.claude/skills/release/SKILL.md`, to extend for tap inclusion).
- **`specflow check --project` strictness** : define if absence of `.specflow/config.yml` is ⚠ warn
  or ❌ fail. Decision : ⚠ warn (config is optional until user wants sync).
- **`--force` edge case** : what if `.specflow.bak` itself is tracked by git and committed ?
  Decision : the `.gitignore` template already embeds dist/ etc. ; add `*.specflow.bak` to
  `.gitignore` bundled via root templates.
